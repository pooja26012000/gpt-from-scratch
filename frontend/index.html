How it actually works

The same core mechanism as GPT — just trained on move notation instead of prose.

1

Custom BPE tokenizer

15,000 real Lichess games, headers stripped, compressed into 1,534 learned tokens via byte-pair encoding built from scratch — no tiktoken, no pretrained vocab.

2

Embeddings

Each token gets a learned vector for what it is, plus a learned vector for where it sits in the sequence.

3

Causal self-attention

Four heads, each computing softmax(QKᵀ/√d)·V by hand — every token attends only to itself and earlier tokens, never the future.

4

Feed-forward + residuals

Four stacked transformer blocks, each with a per-token MLP, Pre-LN layer norm, and residual connections — exactly like GPT-2's architecture, at 1/100th the scale.

5

Sampling

Greedy, temperature, top-k, and top-p decoding — each trading determinism for variety differently, measured for legal-move rate against a real chess rules engine.

Built from scratch — PyTorch model, FastAPI backend on Cloud Run, this page on Vercel.
